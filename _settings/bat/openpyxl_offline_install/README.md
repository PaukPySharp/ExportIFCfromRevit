# Оффлайн-установка openpyxl из архива

Эта папка содержит крайний вариант установки `openpyxl` — напрямую из ZIP-архива,
без использования `pip` и без доступа к интернету.

## Структура

- `install_openpyxl_zip.bat` — основной bat-скрипт.
- `_install_openpyxl_zip.py` — вспомогательный Python-скрипт
  (распаковывает архив в `site-packages` текущего Python).
- В корне проекта должен быть файл: `_for_python/openpyxl.zip`.

## Что делает `install_openpyxl_zip.bat`

1. Определяет установленный Python 3 (через `py -3` или другую настройку в шапке).
2. Проверяет:
   - что существует архив `_for_python/openpyxl.zip`;
   - что рядом лежит `_install_openpyxl_zip.py`.
3. Запускает Python-скрипт, который:
   - находит актуальный каталог `site-packages`;
   - создаёт его при необходимости;
   - распаковывает `openpyxl.zip` в `site-packages`
     с базовой проверкой безопасности (без выхода за пределы каталога).

По завершении в консоли видно:
- `[OK]` — если распаковка прошла успешно;
- `[ОШИБКА] ...` — если архив не найден, не удалось определить `site-packages`
  или произошла ошибка при распаковке.

## Когда использовать

- Нет доступа к интернету.
- Основной скрипт `install_openpyxl_pip_or_whl.bat` не смог установить `openpyxl`
  ни через `pip`, ни из локальных `.whl`.
- По инструкции/рекомендации ответственного за проект.

В обычном сценарии достаточно использовать `install_openpyxl_pip_or_whl.bat`.
Этот офлайн-вариант — резервный инструмент «на крайний случай».

## Как убедиться, что `openpyxl` установлен

После успешного запуска `install_openpyxl_zip.bat` можно проверить установку так:

В командной строке выполните одну из команд:

```bash
py -3 -m pip show openpyxl
```
или
```bash
py -3 -c "import openpyxl; print(openpyxl.__version__)"
```

Если ошибок нет и выводится информация о пакете и его версии — значит, `openpyxl` установлен корректно.
